# Security Policy

## Reporting a Vulnerability

We take the security of The Sound Studio seriously. If you believe you have found a security vulnerability, please report it to us by opening a GitHub Issue with the "Security" label or contacting the maintainer directly.

### Guidelines

- Do not disclose the vulnerability publicly until it has been addressed.
- Provide a detailed description of the vulnerability and steps to reproduce it.
- We will acknowledge receipt of your report and provide an estimated timeline for a fix.

## Supported Versions

Only the latest version of The Sound Studio is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Sensitive Information

We strive to ensure that no private or sensitive information is disclosed in the codebase. If you find any hardcoded credentials or private data, please report it immediately.
